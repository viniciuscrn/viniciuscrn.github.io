Arquiteturas 32 e 64 bits

Disciplina: Administração de Sistemas Proprietários
Turma: Redes — 2º ano
Duração: 2 aulas

1. O que é arquitetura de bits?

A arquitetura de bits de um processador define o tamanho das palavras de dados que ele processa internamente — ou seja, quantos bits ele consegue operar de uma só vez. Isso inclui o tamanho dos registradores, do barramento de endereços e do barramento de dados.

Essa característica afeta diretamente três pilares:

  • A quantidade de memória RAM endereçável;
  • A precisão dos cálculos realizados;
  • A velocidade do processamento de dados.

Analogia: Pense em registradores como caixotes de transporte. Um processador de 32 bits carrega 32 itens por viagem; o de 64 bits carrega 64. Com mais capacidade por viagem, o trabalho termina mais rápido.

2. CPU: diferenças de hardware

2.1 Comparação entre x86 (32 bits) e x86-64 (64 bits)

Característica 32 bits (x86) 64 bits (x86-64)
Tamanho dos registradores 32 bits 64 bits
RAM máxima endereçável 4 GB Teórico: 16 EB
Espaço de endereços 2³² ≈ 4 bilhões 2⁶⁴ (enorme)
Registradores de uso geral 8 16
Extensão de memória (PAE) Até 64 GB* Não se aplica
Paralelismo de instruções Menor Maior

* O PAE (Physical Address Extension) permite que processadores x86 acessem mais memória física, mas cada processo individual ainda fica limitado a 4 GB.

2.2 O limite dos 4 GB no x86

O barramento de endereços de 32 bits gera exatamente 2³² = 4.294.967.296 posições de memória. Contudo, parte desse espaço é reservada para dispositivos de hardware (BIOS, placas de vídeo, chipset), reduzindo a RAM utilizável para aproximadamente 3,2 a 3,5 GB na prática.

Processadores 64 bits duplicaram o número de registradores de uso geral (de 8 para 16 no x86-64), o que reduz os acessos à memória durante cálculos intensivos, melhorando a performance geral.

3. Sistemas Operacionais: edições 32 vs 64 bits

O SO precisa ser compilado para a arquitetura alvo. Um sistema operacional 64 bits explora todos os recursos do processador, enquanto um SO 32 bits instalado em hardware 64 bits desperdiça capacidade e impõe os mesmos limites de RAM.

3.1 Windows — versões e suporte

  • O Windows XP foi o primeiro a ter ampla adoção em versão 64 bits (2005).
  • O Windows 10/11 64 bits é o padrão atual do mercado.
  • O Windows 11 abandonou completamente o suporte a 32 bits — é exigido hardware 64 bits com TPM 2.0.
  • Versões 32 bits do Windows suportam no máximo 4 GB de RAM; versões 64 bits chegam a 128 GB (Home) e 2 TB (Pro/Enterprise).

3.2 Linux — compatibilidade e distribuições

  • Distribuições modernas como Ubuntu 20.04+ eliminaram as ISOs 32 bits para desktop.
  • O kernel Linux 64 bits suporta até 128 TB de memória física em arquitetura x86-64.
  • Ainda existem imagens 32 bits para hardware legado, IoT e sistemas embarcados.

3.3 Modo de compatibilidade e subsistemas

SOs 64 bits implementam camadas de compatibilidade para executar software antigo:

  • Windows: o subsistema WoW64 (Windows on Windows 64) permite executar aplicativos 32 bits em um SO 64 bits sem virtualização.
  • Linux: as bibliotecas lib32 cumprem papel equivalente, permitindo rodar binários 32 bits em sistemas 64 bits.

4. Aplicativos: impactos e escolhas

Um aplicativo precisa ser compilado para a arquitetura alvo. Um binário 32 bits roda em sistemas 64 bits (via compatibilidade), mas o inverso não é possível sem emulação.

4.1 Comparação entre apps 32 e 64 bits

Característica App 32 bits App 64 bits
Memória máxima por processo 2 GB ~128 TB (teórico)
Compatibilidade x86 e x64 Somente x64
Registradores disponíveis 8 16
Uso atual Legado / embarcado Padrão moderno

4.2 Quando a migração para 64 bits faz mais diferença?

Aplicativos que se beneficiam mais da migração são aqueles que processam grandes volumes de dados:

  • Editores de vídeo e áudio;
  • Bancos de dados de grande porte;
  • Servidores de aplicação e web;
  • Máquinas virtuais (Hyper-V, VMware, VirtualBox);
  • Ferramentas científicas e de simulação.

Por que alguns apps ainda usam 32 bits? Compatibilidade com sistemas legados, menor consumo de memória (ponteiros 32 bits ocupam menos espaço) e código antigo sem manutenção ativa são as razões mais comuns.

5. Desempenho e casos de uso

5.1 Quando 64 bits faz diferença real?

  • Edição de vídeo e renderização 3D — precisam de mais de 4 GB de RAM;
  • Bancos de dados de grande porte — indexação de conjuntos massivos de dados;
  • Virtualização (Hyper-V, VMware) — cada VM aloca memória separada;
  • Compilação de projetos grandes — GCC, LLVM se beneficiam de mais memória;
  • Servidores web de alto tráfego — muitos processos simultâneos em memória.

5.2 Quando a diferença é mínima?

  • Navegação web simples e edição de texto;
  • Sistemas com até 2 GB de RAM instalados;
  • Dispositivos embarcados com firmware específico (roteadores, switches);
  • Aplicações de controle industrial com hardware fixo.

6. Resumo: o que muda em cada camada

Camada 32 bits → 64 bits
CPU +8 registradores, barramento de endereços duplicado
Memória endereçável 4 GB → teoricamente ilimitado
Sistema Operacional Gerenciamento de memória expandido
Drivers Devem ser recompilados para 64 bits
Aplicativos Processo pode usar mais de 2 GB de RAM
Segurança ASLR mais eficaz, DEP/NX obrigatório
